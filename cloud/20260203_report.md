### 1. Secure Multi-User Linearly-Separable Distributed Computing
- **精简摘要**：本研究提出了一种新的多用户线性可分分布式计算框架，通过将计算问题转化为稀疏矩阵分解问题，实现了低成本下的大规模并行化增益。为确保数据安全，采用信息论保密框架，提出了两个保密性必要充分条件，并设计了一种通用方案，适用于有限和非有限域。该方案在保证性能的同时，为有限域提供了完美的信息论保密性，在实数域中通过增加高斯共享随机性的方差，可以将互信息界限降至任意小。
- **PDF链接**：[ArXiv下载](https://arxiv.org/pdf/2602.02489v1.pdf)

### 2. Flow Policy Gradients for Robot Control
- **精简摘要**：本研究展示了如何将流匹配策略梯度这一新框架应用于机器人控制策略的训练和微调，该框架无需计算概率密度。通过改进目标函数，成功应用于腿部运动、人形动作追踪和操纵任务，并在两个仿人机器人上实现了稳健的模拟到现实转移。研究还提供了训练动态的消融分析和分析，结果表明策略在从零开始训练时能够利用流表示进行探索，并在微调时相比基线方法具有更好的鲁棒性。
- **PDF链接**：[ArXiv下载](https://arxiv.org/pdf/2602.02481v1.pdf)

### 3. SPARKLING: Balancing Signal Preservation and Symmetry Breaking for Width-Progressive Learning
- **精简摘要**：渐进学习（PL）通过逐步增加模型规模来减少预训练计算开销。尽管已有工作广泛探索深度扩展，但宽度扩展研究不足，且现有方法仅限于训练早期阶段。本文提出SPARKLING框架，通过RMS尺度一致性实现信号保持，稳定扩展过程中的激活统计；通过不对称优化器状态重置和学习率重新预热打破梯度对称性，促进特征多样性。在专家混合模型上的实验表明，SPARKLING在多个宽度轴和优化器家族中均优于从头开始训练，并在2倍宽度扩展下将训练成本降低最多35%。
- **PDF链接**：[ArXiv下载](https://arxiv.org/pdf/2602.02472v1.pdf)

### 4. Stabilizing Decentralized Federated Fine-Tuning via Topology-Aware Alternating LoRA
- **精简摘要**：本文提出了一种名为TAD-LoRA的拓扑感知去中心化低秩适应框架，用于解决在动态通信图下，去中心化联邦学习（DFL）中因LoRA更新的分散聚合导致的训练不稳定问题。通过控制LoRA因子的更新和混合，TAD-LoRA能够控制客户端间的错位。在非凸目标下，理论证明了TAD-LoRA的收敛性，并明确了由交替训练的切换间隔所控制的拓扑引起的交叉项误差与块坐标表示偏差之间的权衡。在不同通信条件下的实验验证了我们的分析，表明TAD-LoRA在各种通信场景下均能保持稳健性能，在强连通拓扑中保持竞争力，在中等和弱连通拓扑中表现出明显优势，特别是在MNLI数据集上取得了优异结果。
- **PDF链接**：[ArXiv下载](https://arxiv.org/pdf/2602.00451v1.pdf)

### 5. A Serverless Edge-Native Data Processing Architecture for Autonomous Driving Training
- **精简摘要**：本文介绍Lambda框架，一种边缘原生平台，通过用户定义函数实现车载数据过滤和处理。该框架采用无服务器抽象层，将应用逻辑与底层执行细节（如调度、部署和隔离）分离。通过将函数即服务（FaaS）原则应用于资源受限的汽车环境，开发者可实现模块化、事件驱动的过滤算法，同时保持与ROS 2和现有数据记录管道的兼容性。在NVIDIA Jetson Orin Nano上评估后，结果显示Lambda框架具有竞争力的性能，减少了延迟和抖动，证明了基于Lambda的抽象能够支持嵌入式自动驾驶系统的实时数据处理。
- **PDF链接**：[ArXiv下载](https://arxiv.org/pdf/2601.22919v1.pdf)

### 6. KUBEDIRECT: Unleashing the Full Power of the Cluster Manager for Serverless Computing
- **精简摘要**：KUBEDIRECT是一种基于Kubernetes的FaaS集群管理器，旨在解决消息传递瓶颈问题。通过利用FaaS平台的共性，实现直接消息传递以提高效率，同时采用新颖的状态管理方案确保一致性。KUBEDIRECT与Kubernetes兼容，每个控制器增加约150行代码。实验表明，其服务延迟比Knative降低26.7倍，性能与最先进的Dirigent平台相当。
- **PDF链接**：[ArXiv下载](https://arxiv.org/pdf/2601.19160v1.pdf)
